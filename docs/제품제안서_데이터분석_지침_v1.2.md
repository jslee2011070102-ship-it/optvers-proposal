## 제0원칙: 절대적 사실성 (Principle of Absolute Factual Integrity)

모든 분석의 최상위 원칙이며, 다른 모든 규칙에 우선한다.

**환각(Hallucination) 절대 금지**
업로드된 데이터 파일에 명시적으로 존재하지 않는 수치, 순위, 상품명, 비율을 추측하거나 창작하지 않는다.
예) "고체탈취제 상품이 2개뿐"처럼 데이터에 없는 숫자를 추정으로 채우는 행위 → 분석 실패로 간주.

**'수치'와 '판단'의 엄격한 분리**
모든 분석은 두 단계로 명확히 분리되어야 한다.
- **수치** = 데이터 파일에서 직접 읽힌 것
- **판단** = 수치로부터 도출한 해석

판단은 반드시 100% 검증된 수치에만 근거할 수 있다.

**불확실성 명시 의무**
데이터에 없거나 추가 확인이 필요한 경우, 추정으로 메우는 대신 아래 표기 중 하나를 반드시 사용한다.

| 상황 | 표기 방식 |
|---|---|
| 데이터 파일에 해당 수치 없음 | `[데이터 없음 — 직접 확인 필요]` |
| 수치는 있으나 해석이 불확실함 | `[추정 — 근거: OOO]` |
| 데이터 범위 밖의 주장 | `[데이터 범위 초과 — 해당 파일로 검증 불가]` |

**임무 실패 조건**
단 하나의 수치라도 데이터 출처 없이 사용되거나, 데이터와 다르게 왜곡된 경우 해당 섹션 전체를 재작성한다.

## 단계 0: 데이터 파일 목록 선언 (Data Source Declaration)

분석 시작 전, 사용 가능한 데이터 파일 전체를 아래 형식으로 먼저 선언한다.
이후 모든 수치는 반드시 이 목록 중 하나를 출처로 명시해야 한다.

### 쿠팡 데이터 (제품 분석 중심)

```
[파일A] 키워드 분석 파일 (Coupang Data)
  포함 데이터: 카테고리월매출, 카테고리월판매량, 배송비율, 리뷰분포, 포화도
               상위상품 순위/가격/리뷰수/판매량/매출/전환율/배송방법
               (참고: 쿠팡은 키워드별 검색량 미제공)

[파일B] 카테고리 분석 파일
  포함 데이터: 카테고리 월 총 매출, 카테고리 월 총 판매량,
               배송 비율, 포화도, 리뷰 분포

[파일C] 상품 리스트 파일 (파일A 또는 파일B의 shoppingList 시트)
  포함 데이터: 상품명, 순위, 가격, 리뷰 수, 클릭 수,
               월 판매량, 월 매출, 전환율, 배송방법
```

### 네이버 키워드 데이터 (소비자 수요 분석 중심)

```
[파일D] 네이버 키워드 분석 데이터
  포함 데이터: 월 검색량 (절대 검색량 기준 — 쿠팡 검색량과 무관)
               인기키워드 (월 검색량, 검색 트렌드)
               자동완성 키워드 (연관 검색 패턴)
               키워드별 경쟁강도 (광고단가 기반)
  주의: 네이버 검색량은 전사이트 검색량이므로, 쿠팡에서의 예상 검색량과 다름
```

선언되지 않은 파일의 데이터는 존재하지 않는 것으로 간주한다.

## 데이터 역할 분담: 쿠팡 vs 네이버 (Data Role Separation)

**핵심 원칙**: 쿠팡과 네이버는 다른 질문에 답하는 상이한 데이터 소스

### 🛍️ 쿠팡 데이터 — "실제 판매되는 제품이 뭔가?"

| 분석 항목 | 출처 | 의미 |
|----------|------|------|
| 시장 규모 | 파일B · 월 매출/판매량 | 이 카테고리에서 실제로 얼마나 팔리는가 |
| 제품 구성 | 파일C · 상위 상품명 | 쿠팡에서 실제로 판매되는 제품 유형은 뭔가 |
| 가격대 분포 | 파일C · 가격 | 소비자가 실제로 구매하는 가격대는 어느 정도인가 |
| 제품 특성 | 파일C · 상품명 분석 | 상위 제품들의 공통 특징은 뭔가 (용량, 소재, 기능 등) |
| 경쟁 강도 | 파일A · 포화도 | 쿠팡에서의 실제 경쟁 상황은 어떤가 |
| 판매 성과 | 파일C · 리뷰수/판매량/전환율 | 실제로 팔리고 있는 제품의 성과는 어떤가 |

**쿠팡 데이터로 답할 수 없는 질문**: 이 카테고리에 대한 소비자의 관심도가 몇 배나 되는가? 소비자들이 뭘 찾고 있는가?

### 🔍 네이버 데이터 — "소비자가 뭘 찾고 있는가?"

| 분석 항목 | 출처 | 의미 |
|----------|------|------|
| 카테고리 관심도 | 파일D · 월 검색량 | 소비자가 이 카테고리에 얼마나 관심이 있는가 |
| 수요 크기 | 파일D · 월 검색량 (절대값) | 전체 시장에서 소비자 수요는 얼마나 큰가 |
| 소비자 니즈 | 파일D · 자동완성 키워드 | 소비자들이 실제로 검색하는 속성/특징은 뭔가 |
| 연관 키워드 | 파일D · 연관 검색 패턴 | 이 카테고리와 관련해서 소비자들은 뭘 함께 관심 갖는가 |
| 트렌드 패턴 | 파일D · 월 검색 변화 | 최근 소비자 관심이 증가/감소하는 방향은 어떤가 |

**네이버 데이터로 답할 수 없는 질문**: 이 제품이 실제로 쿠팡에서 팔리는가? 가격은 얼마인가? 경쟁은 얼마나 심한가?

### 🤝 통합 분석 예시

```
네이버 데이터: "마스크" 검색량 71,600회/월 (소비자 관심 높음)
           + "성인용품" 카테고리와 함께 검색되지 않음 (관심도 낮음)
쿠팡 데이터:  상위 제품명에 "마스크" 제품 없음
           + 대신 바이브레이터, 콘돔 등만 판매 중
결론:        71,600회 검색량은 "페이스 마스크"이며, 성인용품 카테고리의
           실제 수요가 아님 → "카테고리 오염" 진단
```

## 단계 1: 원자 단위 데이터 추출 (Raw Data Extraction)

분석 시작과 동시에 업로드된 모든 파일을 컬럼 단위로 분해하여 아래 4개 카테고리로 분류한다.
이 단계에서는 어떤 해석도 가미하지 않으며 순수 수치와 텍스트만 기록한다.

### ① 시장규모 데이터 (Market Size)
쿠팡: 카테고리 월 매출, 월 판매량
네이버: 카테고리 월 검색량 (소비자 관심도의 대리지표)
→ 이 카테고리의 시장이 얼마나 크고 수요 있는지를 보여주는 원재료.

### ② 경쟁/포화도 데이터 (Competition)
쿠팡: 포화도(매출/판매/리뷰), 배송 방식 비율, 상위 상품 전환율
네이버: 키워드별 경쟁강도 (광고단가)
→ 시장 내 경쟁 강도와 진입 난이도를 보여주는 원재료.

### ③ 소비자 니즈 데이터 (Consumer Demand)
네이버: 자동완성 키워드, 연관 검색 패턴
쿠팡: 상위 제품명 분석 (실제 판매되는 제품의 특징)
→ 소비자들이 실제로 찾고 있는 제품 유형을 보여주는 원재료.

### ④ 제품 분석 데이터 (Product Composition)
쿠팡: 상위 상품명, 가격대, 리뷰 패턴, 판매량
→ 실제로 팔리는 제품의 특성, 가격대, 소구 포인트를 보여주는 원재료.

**출처 표기 규칙**
모든 수치 뒤에는 반드시 `[출처 · 컬럼명]` 형식으로 출처를 표기한다.

```
예) 월 검색량 71,600회 [네이버 · 월 검색량]
    카테고리 월 매출 59.8억 [쿠팡 · 월간 총 매출]
    상위 제품 평균 가격 32,000원 [쿠팡 · 가격]
    경쟁강도 높음 [네이버 · 경쟁강도]
```

## 단계 2: 3단계 해석 프로세스 (Interpretation Process)

단계 1에서 추출된 수치만을 근거로 아래 3단계 순서로 해석을 진행한다.
각 단계의 판단 기준은 사전에 정의된 기준값과 대조하며, 기준값이 없는 항목은 판단 보류로 처리한다.

---

### Step 1 — 시장 확인: "소비자 수요가 충분한가?"

**네이버 검색량 기준** (소비자 수요의 절대량)

| 항목 | 출처 | ◎ 기준 | △ 기준 | ✕ 기준 |
|---|---|---|---|---|
| 월 검색량 (네이버) | 파일D · 월 검색량 | 5,000회+ | 1,000~4,999회 | 999회- |
| 검색 트렌드 | 파일D · 월 검색 변화 | 증가/안정 | 변동 | 감소 추세 |

**쿠팡 시장규모 확인** (수요가 실제로 구매로 전환되는가)

| 항목 | 출처 | ◎ 기준 | △ 기준 | ✕ 기준 |
|---|---|---|---|---|
| 카테고리 월 매출 | 파일B · 월간 총 매출 | 30억+ | 10~29억 | 10억- |
| 카테고리 월 판매량 | 파일B · 월간 총 판매량 | 5만개+ | 1만~4.9만개 | 1만개- |

**Step 1 통과 기준**:
- 네이버 월 검색량 ◎ (5,000회+) 이상
- AND 쿠팡 시장규모 2개 항목 중 1개 이상 ◎

→ 소비자 관심(네이버) + 실제 거래량(쿠팡) 동시 확인

---

### Step 2 — 경쟁 구조: "들어갈 틈이 있는가?"

**쿠팡 포화도 기준** (실제 경쟁 강도)

| 항목 | 출처 | ◎ 기준 | △ 기준 | ✕ 기준 |
|---|---|---|---|---|
| 1위 매출 포화도 | 파일C · 1위 매출 포화도 | 20% 미만 | 20~35% | 35% 초과 |
| 1~3위 매출 포화도 | 파일C · 1-3위 매출 포화도 | 50% 미만 | 50~70% | 70% 초과 |
| 1~3위 리뷰 포화도 | 파일C · 1-3위 리뷰 포화도 | 40% 미만 | 40~60% | 60% 초과 |
| 로켓 계열 배송 비율 | 파일C · 로켓+판매자로켓 비율 합산 | 70% 미만 | — | 80% 이상 (로켓 등록 필수) |
| 상위 5개 평균 전환율 | 파일C · 전환율 (상위 5개 평균) | 10%+ | 5~9% | 5% 미만 |

**Step 2 통과 기준**: 매출 포화도 2개 항목(1위/1~3위)이 모두 ◎이면 통과.
단, 리뷰 포화도 △·✕, 로켓 비율 ✕ 항목은 "진입 전제 조건"으로 별도 명시.

---

### Step 3 — 소비자 니즈와 공백 발견: "제품 방향을 특정할 수 있는가?"

이 단계는 **네이버 키워드 중심**의 패턴 분석. 소비자가 실제로 찾는 것은 뭔가?

#### 3-1단계: 네이버 자동완성 분석 (Consumer Intent)

| 항목 | 출처 | 분석 방법 | 니즈 판단 기준 |
|---|---|---|---|
| 자동완성 키워드 패턴 | 파일D · 자동완성키워드 | 속성/특징으로 분류 (소재, 용량, 기능, 타겟 등) | 특정 속성에 3개 이상 등장 = 강한 니즈 신호 |
| 연관 검색 키워드 | 파일D · 연관 검색 패턴 | 카테고리와 함께 검색되는 키워드 | "성인용품"과 함께 검색 vs 단독 검색 |
| 검색량 상위 키워드 | 파일D · 월 검색량 순위 | 상위 10개 키워드의 공통 특징 추출 | 반복되는 속성/특징 |

#### 3-2단계: 쿠팡 공급 분석 (Supply vs Demand)

| 항목 | 출처 | 분석 방법 | 공백 판단 기준 |
|---|---|---|---|
| 네이버 고검색 키워드 vs 쿠팡 상품 | 파일D + 파일C | 네이버에서 검색 많은 키워드 → 쿠팡 상품명에 미포함 카운트| 검색량 5,000+ 인데 상품명 포함 3개 이하 |
| 상위 제품 공통 속성 | 파일C · 상품명 | 상위 5개 상품명의 공통 특징 추출 | 3개 이상에서 공통으로 부재한 속성 |
| 가격 공백대 | 파일C · 가격 | 5,000원 구간별 상품 수 정리 | 상품 수 0~1개인 가격대 |

**Step 3 통과 기준**:
- 네이버 고검색 키워드 1개 이상 (쿠팡 상품 부족)
- AND 자동완성 패턴 2개 이상 (소비자 니즈 명확)
- AND 상위 제품 공통 속성 1개 이상 (시장 구성 파악)

→ 합산: "소비자는 OOO를 찾는데, 쿠팡에는 OOO 제품이 부족하다" 진단

---

## 단계 3: 최종 판정 및 전제 조건 명시 (Final Verdict)

3개 Step 결과를 종합하여 아래 4가지 판정 중 하나를 선택한다.
판정문에는 반드시 각 Step의 결과 요약과 데이터 출처가 함께 기재되어야 한다.

| 판정 | 조건 |
|---|---|
| **적극 추천** | Step 1 통과 (네이버 수요 + 쿠팡 거래) + Step 2 통과(경쟁 진입가능) + Step 3 명확 + 검증 에이전트 통과 |
| **조건부 추천** | Step 1 통과 + Step 2 통과(전제 조건) + Step 3 명확 + 검증 에이전트 통과 |
| **틈새 제안** | Step 1 통과 + Step 2 포화도 △ + Step 3 롱테일 키워드 공백 + 검증 에이전트 통과 |
| **재검토** | Step 1 미통과 또는 Step 2 포화도 ✕ 또는 Step 3 불명확 또는 검증 에이전트 불합격 |

**전제 조건 명시 규칙**
"조건부 추천" 판정 시, Step 2에서 △ 또는 ✕로 판정된 항목은 반드시
"이 조건을 충족해야 진입 가능하다"는 문장으로 별도 명시한다.

---

## 단계 4: 제품 방향 특정 (Product Direction)

네이버 소비자 니즈 + 쿠팡 공급 현황을 바탕으로 제품 방향을 특정한다.
모든 항목의 근거는 반드시 데이터에서 인용하며, 데이터에 없는 항목은 `[데이터 없음]`으로 표기한다.

```
제품 제안 근거:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
소비자 니즈 (네이버)
- "OOO" 검색량: N,NNN회/월 [네이버 · 월 검색량]
- 자동완성 키워드: "OOO", "OOO", "OOO" [네이버 · 자동완성]
- 연관 검색: "OOO" (카테고리와 함께 검색) [네이버 · 연관검색]

공급 현황 (쿠팡)
- "OOO" 검색 키워드 상품: N개만 존재 [쿠팡 · 상품 카운트]
- 상위 제품 공통 특징: OOO, OOO [쿠팡 · 상위 상품명]
- 가격대 공백: OOO~OOO원 구간 상품 N개 [쿠팡 · 가격 분포]

제품명 제안: [소재] + [카테고리] + [핵심특성]
예시: "실리콘 바이브레이터 10단계 진동 방수형" [네이버 니즈 기반, 쿠팡 공급 부족]

제조 방식: [데이터 없음 — 고객사 확인 필요]
```

> **⚠️ 주의**: 제조 방식, 원가, ODM 업체 정보는 데이터 파일에 존재하지 않으므로
> 반드시 `[데이터 없음]` 표기 후 고객사 확인 사항으로 처리한다.

## 단계 5: 예상 매출 시나리오 (Revenue Scenario)

매출 시나리오는 데이터 기반 수치와 가정값을 반드시 구분하여 표기한다.

### 데이터 기반 수치
- 카테고리 월 총 매출 `[쿠팡 · 월간 총 매출]`
- 카테고리 월 총 판매량 `[쿠팡 · 월간 총 판매량]`
- 상위 5개 평균 전환율 `[쿠팡 · 전환율]`
- 상위 상품 가격대 분포 `[쿠팡 · 가격]`
- 소비자 수요 크기 `[네이버 · 월 검색량]`

### 가정값 (데이터 없음 — 반드시 명시)
- 신제품 목표 점유율: `[가정값 — 고객사와 협의 필요]`
- 판매대행 수수료율: `[가정값 — 계약 조건에 따라 상이]`
- 초기 리뷰 확보 비용: `[가정값 — 마케팅 예산에 따라 상이]`

### 시나리오 작성 규칙

보수·중간·낙관 3가지 시나리오를 작성하되,
각 시나리오의 판매량 가정 근거를 반드시 명시한다.

```
보수적: 카테고리 월 판매량 [쿠팡]의 OO% 점유 가정 → OOO개
중간:   카테고리 월 판매량 [쿠팡]의 OO% 점유 가정 → OOO개
낙관적: 카테고리 월 판매량 [쿠팡]의 OO% 점유 가정 → OOO개

수수료: [가정값 OO% — 실제 계약 조건으로 교체 필요]
```

## 단계 6: 이중 검증 시스템 (Dual Validation System)

제안서 제출 전 두 단계의 검증을 반드시 통과해야 한다.

---

### 6-1단계: 자기검증 (Self-Check)

작성자가 수행하는 데이터 무결성 검증. 아래 체크리스트를 반드시 통과해야 한다.
하나라도 ✕이면 해당 섹션을 재작성한다.

```
✅ 검증 1 — 출처 없는 수치 스캔
   모든 수치에 [출처 · 컬럼명] 출처가 표기되어 있는가?
   데이터 파일에 없는 수치가 출처 없이 사용되지 않았는가?

✅ 검증 2 — 쿠팡 vs 네이버 데이터 혼용 확인
   네이버 검색량을 쿠팡 판매량과 혼동하지 않았는가?
   쿠팡 데이터와 네이버 데이터의 역할이 명확히 구분되어 있는가?

✅ 검증 3 — 수치/판단 분리 확인
   "데이터에서 읽힌 것"과 "도출한 판단"이 혼재되지 않는가?
   판단 문장 앞에 근거 수치가 먼저 제시되어 있는가?

✅ 검증 4 — 불확실성 표기 확인
   데이터에 없는 항목(제조방식, 수수료율 등)에 [데이터 없음] 표기가 되어 있는가?
   추정으로 채워진 빈칸이 없는가?

✅ 검증 5 — Step 3 분석 근거 완결성
   네이버 소비자 니즈와 쿠팡 공급 공백이 명확히 입증되어 있는가?
   "소비자는 OOO를 찾는데, 쿠팡에는 부족하다"는 로직이 성립하는가?

✅ 검증 6 — 판정 근거 완결성 확인
   최종 판정에 Step 1~3 결과 요약이 모두 포함되어 있는가?
   "조건부 추천" 판정 시 전제 조건이 명시되어 있는가?
```

---

### 6-2단계: 검증 에이전트 (Validation Agent) ★ 권장

**목적**: 제안된 제품이 실제로 소비자가 찾는 것인지, 카테고리 적합한지 독립적으로 검증

제안 문서 작성 완료 후, **독립적인 검증 에이전트**가 다음을 평가한다:

#### 검증 에이전트가 평가하는 항목

1. **카테고리 적합성**
   - 네이버와 쿠팡에서 함께 나타나는 수요인가?
   - 네이버에서는 많이 검색되지만 쿠팡에서는 판매되지 않는 건 아닌가?
   - 예시 불합격: 성인용품에서 "페이스 마스크" (네이버 검색량 높지만 쿠팡과 무관)

2. **실제 구매 가능성**
   - 이 카테고리의 고객이 정말 이 제품을 사겠는가?
   - 단순 논리적 가능성이 아닌 **현실의 쿠팡 판매 데이터와 일치하는가?**
   - 네이버 수요와 쿠팡 공급이 매칭되는가?

3. **제품명 논리 검증**
   - 제품명이 논리적으로 성립하는가?
   - 서로 다른 제품 카테고리 키워드가 결합되지 않았는가?
   - 브랜드명이 포함되지 않았는가?
   - 마케팅 문구가 제품명으로 사용되지 않았는가?

4. **키워드 카테고리 분류 검증**
   - 각 제안 제품이 단일 카테고리 키워드에서만 도출되었는가?
   - 서로 다른 제품 카테고리의 키워드를 강제로 결합하지 않았는가?

#### 검증 에이전트의 판정 규칙

```
✅ 합격 (PASS)
   - 네이버 수요 + 쿠팡 공급 매칭 명확
   - 제안 제품이 카테고리 적합
   - 고객의 실제 구매 가능성이 높음
   - 제품명이 논리적으로 성립
   - 제품 카테고리가 혼용되지 않음

❌ 불합격 (FAIL)
   - 네이버 검색량 높지만 쿠팡과 무관한 제품
   - 카테고리 외부 제품 제안
   - 제품명에 브랜드명 포함
   - 서로 다른 제품 카테고리 키워드 결합
   - 논리적으로 성립하지 않는 제품명
```

#### 불합격 시 조치

불합격 판정 시 **자동 재생성** 프로세스 실행:
1. 검증 에이전트의 불합격 이유 분석
2. 네이버 소비자 니즈 재분석 + 쿠팡 공급 재조사
3. 더 엄격한 제약 조건으로 제품 재제안
4. 재검증 (최대 2회 재시도)
5. 2회 재시도 후에도 불합격 시 → 수동 개입 필요

---

**권장 사항**: 모든 제안 문서는 **검증 에이전트 통과 여부**를 최종 판정의 필수 요소로 간주한다.
검증 에이전트 통과 없이는 "최종 추천" 판정 불가.

## 부록: 오류 유형 라이브러리 (Error Pattern Library)

분석을 반복할수록 이 목록에 추가한다.

| 오류 유형 | 발생 사례 | 올바른 처리 |
|---|---|---|
| 데이터 혼용 | 네이버 검색량을 쿠팡 판매량처럼 해석 | 네이버 = 소비자 관심도, 쿠팡 = 실제 거래량으로 명확 구분 |
| 카테고리 오염 | "마스크" 71,600회를 성인용품 수요로 오인 | 네이버와 쿠팡 데이터 교차 확인 (쿠팡에 마스크 제품 없음 = 오염) |
| 수치 환각 | "상품이 2개뿐" → 파일에 없는 카운트 사용 | 쿠팡 상품명을 직접 카운트한 결과만 기재 |
| 가정값 무표기 | "판매대행 수수료 15%" → 협의된 적 없는 수치 | `[가정값 OO% — 계약 조건으로 교체 필요]` |
| 판단의 수치화 | "경쟁이 치열하다" → 근거 수치 없음 | "1~3위 포화도 43.1% [쿠팡] → 진입 가능" 구체적 표기 |
| 데이터 범위 초과 | "소비자 반응이 좋다" → 파일 없음 | `[데이터 없음 — 쿠팡 리뷰 별도 수집 필요]` |
| 카테고리 혼용 제품명 | "마스크+바이브레이터 세트" → 다른 카테고리 | 단일 카테고리만 제안. 검증 에이전트에서 불합격 |
| 브랜드명 포함 | "코멧 스타일 마스크" → 경쟁사명 사용 | 브랜드명 제거, 제품 특성으로 대체 |
| 자기합리화 | 논리적으로 이상한 제안을 정당화 | 검증 에이전트의 독립적 평가에 의존, 불합격 시 재생성 |
| 네이버 검색량 오용 | 쿠팡 전체 매출을 네이버 검색량으로 추정 | 네이버와 쿠팡 데이터의 용도 구분: 네이버=수요 신호, 쿠팡=실제 거래 |
